# Workshop Tools

## What this is

A small, growing collection of lightweight facilitation tools built for real workshops. Each tool solves a specific moment in a session — checking in, reading the room, managing ideas that come up at the wrong time, and making sense of a wall full of sticky notes.

This toolkit started as a vibe coding learning project, built by a lead digital product designer and expert facilitator who wanted to explore what AI-assisted development could produce in a professional practice context. The tools are intentionally simple, browser-based, and designed to run without setup on any device.

---

## The tools

**Energy Check-In**
A quick pulse check before the session gets going. Participants share how they're feeling, and the facilitator gets a live read of the room's energy level — displayed as a word cloud so patterns surface immediately.

**Temperature Check**
Measures how aligned or divided the group is on a topic or decision. When results come back polarised or low, the tool suggests AI-generated facilitation moves to help the facilitator respond in the moment.

**Parking Lot**
A place to capture questions, ideas, and tangents that matter but don't belong right now. Participants can add items during the session, and the facilitator can return to them before closing.

**Synthesis Accelerator**
Paste in sticky notes from a brainstorm or research session and let AI cluster them into themes. The tool also generates "How Might We" questions from each theme — a useful jumping-off point for ideation or prioritisation.

## How to use

Open `index.html` in a browser to access the tool launcher. Each tool runs entirely in the browser with no installation required.

For tools with a facilitator view and a participant view, the facilitator controls the session from their screen while participants join on their own devices. Check each tool's interface for instructions — they're designed to be self-explanatory in the room.

The Synthesis Accelerator requires an Anthropic API key to run AI clustering. You'll be prompted to enter it when you open the tool.

---

## How to contribute

New tools are welcome, but every addition starts with a completed vibe coding brief before any building begins. The brief captures the facilitation moment the tool is designed for, what the facilitator needs to do or see, what participants experience, and any constraints worth knowing up front. This keeps the toolkit coherent and makes the actual build faster.

If you have an idea for a tool, write the brief first. Then build.

Follow the existing file structure — each tool lives in its own folder with an `index.html` as the entry point. Keep the visual style consistent with the existing tools (see `index.html` in the root for the design tokens).
